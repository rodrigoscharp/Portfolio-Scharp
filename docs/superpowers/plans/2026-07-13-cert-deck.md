# Certifications Card Deck Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the vertical certificate list inside Education with a horizontal, drag-open "card deck" (5 certificates) plus a separate highlighted block for the university degree, on both `index.html` and `dev.html`.

**Architecture:** A new `.cert-deck` component — pointer-driven (not scroll-pinned, not auto-drifting), pure vanilla JS with no GSAP dependency, added to `assets/js/motion.js` and `assets/css/lando-system.css` once, then wired into both pages' Education sections. Each card's rotation is fixed at init; only inter-card spacing (and, once the fan would overflow the container, a compensating pan) is driven by drag progress.

**Tech Stack:** Static HTML/CSS, vanilla JS (Pointer Events). No GSAP/ScrollTrigger — deliberately independent of the `reduced-motion`/GSAP gate at the top of `motion.js`, because the deck only moves in direct response to the user's own drag (not autopliaying motion), same category as native scrollbar dragging.

## Global Constraints

- Deck cards drop the `xp-logo` images that index.html currently shows for
  these items (per the approved spec/mockups, which never included logos) —
  applies to **both** pages, so the new markup renders identically on
  index.html and dev.html.
- Rotation and vertical offset are fixed per card at init and **never change**
  during drag — only horizontal spacing (and pan, if needed) responds to
  drag progress. (Per spec: user picked "keeps rotation" over "straightens.")
- No idle auto-drift — the deck never moves unless the user is actively
  dragging it (per spec).
- No vertical-scroll pin — unlike `.projects-track`, dragging never locks
  page scroll.
- Fan angle/spacing formulas must be based on live card count (`cards.length`),
  never a hardcoded number, so adding a 6th certificate later needs no CSS/JS
  changes — only a new `.cert-card` block in the HTML.
- `.superpowers/` stays out of git (already in `.gitignore` — do not commit
  anything under it).

## Design note: reconciling "closed fan" with "container overflow"

The approved spec describes recomputing each card's spacing on drag, but a
literal reading (spacing grows, positions never pan) leaves cards from a
5-card fan positioned beyond the container's right edge with nowhere to
scroll to. This plan resolves it by combining two effects into one
`progress` value (`0` to `1`, driven directly by drag distance):

```
x_i(progress) = i * (closedStep + progress * (openStep - closedStep))
                - progress * panOffset
```

- The first term is the fan spacing growing from tightly overlapped
  (`closedStep`) to fully separated (`openStep`) — exactly what the spec
  describes.
- The second term (`panOffset`, computed from how far the fully-open fan
  would overflow the container) linearly pans everything left as the deck
  opens, so by `progress = 1` the rightmost card is fully visible within the
  container. On desktop, where the whole open fan usually fits,
  `panOffset` comes out to `0` and this term does nothing. On narrow mobile
  viewports, earlier cards scroll off-screen to the left as later ones
  become visible while fully opening — the same trade-off every other
  horizontal gallery on this site already makes (e.g. `.projects-track`).

This is a linear interpolation between two fixed endpoints, so it never
overshoots the container at intermediate drag positions.

- [ ] **Step 2: Add the JS behavior**

Open `assets/js/motion.js`. Find this line near the top (right after the


Insert the new block **between** those two lines (so it runs unconditionally,
independent of `prefers-reduced-motion` and of whether GSAP loaded — it has
no GSAP dependency and only ever moves in direct response to the user's own
drag, not autoplaying motion):

```js
  /* certifications: closed "card deck" that fans open horizontally as the
     user drags — pointer-driven (mouse or touch), not scroll/pin-driven,
     no idle auto-drift. Each card's rotation is fixed at init; only the
     spacing between cards (and, if the open fan would overflow the
     container, a compensating pan) responds to drag progress. */
  document.querySelectorAll('.cert-deck').forEach(deck => {
    const cards = [...deck.querySelectorAll('.cert-card')];
    const n = cards.length;
    if (!n) return;

    const maxSpread = 18;   // total degrees from leftmost to rightmost card
    const closedStep = 26;  // px between consecutive cards, fully closed
    const gap = 22;         // px gap between cards, fully open
    const angleStep = n > 1 ? maxSpread / (n - 1) : 0;
    const meta = cards.map((card, i) => ({
      card,
      angle: (i - (n - 1) / 2) * angleStep,
    }));

    let cardWidth, openStep, panOffset, dragDistance;
    const measure = () => {
      cardWidth = cards[0].offsetWidth;
      openStep = cardWidth + gap;
      const finalRightEdge = (n - 1) * openStep + cardWidth;
      panOffset = Math.max(0, finalRightEdge - deck.clientWidth + 24);
      dragDistance = Math.max(320, openStep * (n - 1) * 0.6);
    };

    let progress = 0;
    const render = () => {
      meta.forEach(({ card, angle }, i) => {
        const x = i * (closedStep + progress * (openStep - closedStep)) - progress * panOffset;
        const y = Math.abs(angle) * 0.6;
        card.style.transform = `translate(${x}px, calc(-50% + ${y}px)) rotate(${angle}deg)`;
        card.style.zIndex = i + 1;
      });
    };

    measure();
    render();

    let dragging = false, startX = 0, startProgress = 0;
    const clamp01 = v => Math.min(1, Math.max(0, v));
    deck.addEventListener('pointerdown', e => {
      dragging = true; startX = e.clientX; startProgress = progress;
      deck.setPointerCapture(e.pointerId);
    });
    deck.addEventListener('pointermove', e => {
      if (!dragging) return;
      progress = clamp01(startProgress + (e.clientX - startX) / dragDistance);
      render();
    });
    ['pointerup', 'pointercancel'].forEach(ev =>
      deck.addEventListener(ev, () => { dragging = false; }));

    window.addEventListener('resize', () => { measure(); render(); });
  });

```
